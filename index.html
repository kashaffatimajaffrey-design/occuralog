The Problem

A mid-size steel fabrication company. Structural components were due Tuesday. It is Thursday. The client is calling.

Call the production floor supervisor.
Check a WhatsApp message from two days ago about a material shortage.
Open an Excel sheet to find which vendor supplies the raw steel.
Call that vendor.
Discover the vendor's truck broke down — and nobody told anyone.
→ That chain took four hours. The answer was a broken truck.

This happens every day across thousands of companies in Pakistan, India, Bangladesh, Egypt, and Nigeria — manufacturers, construction firms, logistics operators who digitised in the worst possible way: Excel for inventory, WhatsApp for communication, paper for approvals. Each tool is an island. Enterprise solved this with SAP twenty years ago. SAP costs millions and is completely inaccessible to this market. Occuralog starts here.

What Is Occuralog

A syncretic library — the right tool, for each layer of the problem.

It ingests what's already happening — WhatsApp exports, Excel files, ERP API calls — structures it into a unified timestamped event log, and makes it queryable. So the question takes seconds, not hours. Everything underneath — concurrent ingestion, legacy parsing, anomaly detection, semantic search — stays invisible to the developer. The API stays clean regardless of how messy the input is.

from occuralog import EventStream stream = EventStream(source="whatsapp_export.txt") stream.ingest() stream.query("why was order #4821 delayed") # → structured answer, not a four-hour phone chain
Architecture

Syncretic by design.

Each layer uses the language built for that job.

EVENT INGESTION Rust Concurrent, append-only, zero GC pauses — compiles to a small binary that runs on cheap hardware.
LEGACY PARSING C++ via FFI Battle-tested libraries for malformed Excel, fixed-width ERP exports, broken EDI — called from Rust.
ORCHESTRATION + ML Python Anomaly detection, LLM extraction, scheduling, API exposure — the glue layer.
STORAGE + SEARCH Postgres + pgvector Structured event log and semantic search over unstructured notes, in one database.

The algorithm

Doesn't change across domains. The extraction rules and schema do — a factory floor today, a hospital ward log tomorrow, a construction site RFI tracker next.

01IngestionConcurrent intake, append-only writeRust
02ParsingNormalise format, handle malformed dataC++ / FFI
03ExtractionUnstructured text → structured JSON eventsPython + LLM
04StorageTimestamped event log + semantic indexPostgres + pgvector
05DetectionAnomaly and pattern recognition over the event streamPython
06QueryNatural language → structured answer—

v0.1 extraction strategy

Start with rules: if a message contains delay, nahi aya, late, pending — classify as a delay event. Captures roughly 60% of cases with zero ML overhead. Everything the rules miss, the LLM handles. Every run is tracked in MLflow so the rule layer and LLM layer improve independently over time.

Long-Term Vision

A library today. A language eventually.

Library
— research phase, now
→
Standalone Language
— multi-year, ahead

The long-term vision is a syncretic language — one that natively expresses the concepts this library currently stitches together across four languages. Designed for operational intelligence the way Julia was designed for scientific computing, or Rust for systems safety. That's a multi-year project. The library is where it starts — you need to know what you wish the language had, before you can design it. Occuralog is the research phase.
